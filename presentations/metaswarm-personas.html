The premise

A single agent that plans, builds, reviews, and ships its own work grades its own homework.

metaswarm splits that one mind into many — each with narrow expertise, a distinct point of view, and the authority to block the others.

Why personas at all

Specialization is a feature, not decoration.

Narrow & deep beats broad & shallow
The Researcher doesn't design. The Designer doesn't implement. Each persona carries one mandate and judges only that.
Adversaries, not collaborators
Reviewers exist to refute, not to agree. A fresh reviewer verifies the contract — never "did they fix what I flagged?"
Gates are blocking, not advisory
Typecheck, lint, tests, coverage, DoD — all are mandatory state transitions. No persona may wave another through.
Complexity hidden from the human
A nine-phase pipeline runs underneath one instruction: "describe what you want."

The keystone rule

The reviewer is always a stranger.

After any FAIL, a brand-new reviewer instance is spawned — no memory of the prior round. This is the single rule that holds in both Team Mode and Task Mode, without exception.

✕ Anchored reviewer

Remembers its earlier notes. Asks: "did they fix the three things I flagged?" Sees three green checks, approves — and never re-checks the contract as a whole. Bias smuggled in as efficiency.

✓ Fresh reviewer

Has never seen this code. Asks: "does this satisfy the Definition of Done?" Verifies every item independently, with file:line evidence. The only thing it can be loyal to is the spec.

Two dispatch mechanisms · identical work product

Team Mode & Task Mode.

Task Mode default · zero overhead

  • Fire-and-forget subagents; full context passed in the prompt.
  • Cold start each spawn — but everything it needs is in-prompt.
  • Sequential handoffs flow through the orchestrator.
  • Parallel work = parallel Task() calls (the 5-agent design panel).
  • Always available. The safe fallback.

Team Mode persistent teammates

  • Named teammates persist and talk directly via SendMessage.
  • Coder keeps context across WU-001 → WU-002 — no cold start.
  • Design reviewers retain context across revisions.
  • Bridge rule: only the orchestrator writes to the durable record.
  • Adversarial reviewer is still a fresh Task() — never a teammate.

Detected once at startup: if TeamCreate + SendMessage exist → Team Mode, else → Task Mode.
